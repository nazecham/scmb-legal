Privacy Policy

Waves  ·  Last updated: May 2026  ·  Operated from Canada  ·  Index

1. What We Collect

Waves collects the minimum data necessary to function:

  • Discord User ID and username — recorded alongside song requests and command usage to attribute activity in the bot's internal dashboard. This is held in memory only and is not written to any database or persistent file.
  • Discord Guild (server) ID and name — used to associate activity with the correct server. Also held in memory only.
  • Song request history — track titles and URLs of songs requested through the bot, stored in memory for dashboard display. This data is cleared when the bot restarts.
  • Command usage — a record of which slash commands were used, by whom, and whether they succeeded. Also in-memory only.

We do not collect message content, voice audio, or any data beyond what is described above.

2. What We Do Not Collect

  • No passwords, email addresses, or payment information
  • No message content outside of slash command arguments
  • No voice audio
  • No data is sold or shared with third parties

3. How Data Is Used

Data collected is used solely to operate the bot — displaying queue history and command stats in the operator dashboard. It is not used for advertising, profiling, or any purpose beyond bot functionality.

4. Data Retention

All user-associated data (user IDs, usernames, song history, command logs) is held in memory only and is permanently deleted whenever the bot restarts. No user data is written to disk.

Operational log files (bot.log, uvicorn.log) may be retained on the host server for debugging purposes. These logs record system events and do not contain personally identifiable information beyond what Discord's platform itself transmits.

5. Third-Party Services

Waves interacts with the following third-party services to function:

  • Discord — the platform the bot operates on. Discord's own Privacy Policy applies to your use of Discord.
  • YouTube / yt-dlp — used to stream audio. No user data is transmitted to YouTube beyond what a normal browser request would include (IP address, etc.).
  • Spotify — playlist and track metadata may be fetched from Spotify's public web endpoints when a Spotify link is provided. No user account data is sent to Spotify.

6. Your Rights (PIPEDA)

As a Canadian-operated service, we follow the Personal Information Protection and Electronic Documents Act (PIPEDA). You have the right to:

  • Know what personal information we hold about you
  • Request correction or deletion of your data

Given that all user data is held in memory only and clears on restart, there is typically no persistent data to access or delete. If you have concerns, contact us via the support server linked in the bot's /about command.

7. Changes to This Policy

We may update this policy from time to time. Continued use of the bot after changes constitutes acceptance of the updated policy. The "last updated" date at the top of this page will reflect any changes.

Questions? Reach out via the support server linked in /about.
